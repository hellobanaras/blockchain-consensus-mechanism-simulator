## 0) Elevator Pitch

“**Consensus Lab**” is a Blazor web app that lets users **spin up virtual nodes**, **choose a consensus protocol**, **run rounds**, and **watch the chain grow** in real time. It provides **analytics, block explorer, and protocol playground** plus optional **Supply-Chain** and **Federated-Learning** payload modes.

---

## 1) User Stories (acceptance criteria)

### US-01: Choose Protocol & Run

* As an Operator, I can select **PoW/PoS/DPoS/PoA/PBFT/PoET/PoB**, set node count, rounds, and latency.
* When I click **Run**, I see **per-round winners, block hashes, validations**, and **final height**.
* AC:

  * At least 5 nodes; 1–100 rounds.
  * Live log stream via SignalR.
  * Simulation stops/pauses/resumes.

### US-02: Block Explorer

* I can browse blocks with fields: `height, hash, prev_hash, proposer, protocol, payload meta, timestamp`.
* AC:

  * Click a block → detail page.
  * Chain tip badge; orphan rate (if mode simulates forks).

### US-03: Analytics

* I can see **winner distribution**, **stake/burn trends**, **vote/quorum stats** (PBFT), **wait times** (PoET).
* AC:

  * Export CSV/PNG.
  * At least 3 charts per protocol.

### US-04: Protocol Playground (config)

* I can edit **per-protocol parameters** (e.g., PoW hash-power, PoS initial stakes, DPoS delegates, PBFT faulty nodes, PoB burn strategy, PoET wait bounds).
* AC:

  * Changes are persisted per experiment.
  * Validation & sensible defaults.

### US-05: Payload Modes

* I can toggle payloads:

  * **Supply-Chain**: product events (manufactured→shipped→delivered).
  * **Federated-Learning**: toy model updates and FedAvg.
* AC:

  * Block payload card renders mode-specific details.
  * Simple accuracy chart for FL.

### US-06: Roles & Auth

* Viewer: read-only.
* Operator: run sims, manage configs.
* Admin: manage users, feature flags.
* AC:

  * Unauthorized requests → 401/403.
  * Audit log for admin actions.

## 4) Consensus Engine Design

### Interfaces

```csharp
public interface IConsensusProtocol
{
    string Name { get; }
    Task InitializeAsync(SimContext ctx, ProtocolParams p, CancellationToken ct);
    Task<RoundResult> RunRoundAsync(int round, CancellationToken ct);
}

public interface IBlockValidator
{
    bool ValidateLink(BlockHeader header, string prevHash);
    bool ValidateProtocolMetadata(string protocol, IReadOnlyDictionary<string,object> meta);
}

public sealed class SimContext
{
    public IList<Node> Nodes { get; }
    public ILedgerService Ledger { get; }
    public IEventBus Events { get; } // emits RoundUpdate for SignalR
    public Random Rng { get; }
}
```

### Pluggable Protocols

* `PowProtocol` (expovariate time to find block; power-weighted)
* `PosProtocol` (stake-weighted choice; compounding reward)
* `DposProtocol` (vote → top-K delegates → round-robin)
* `PoaProtocol` (authority set → round-robin)
* `PbftProtocol` (leader proposes; 2/3 quorum on votes; fault injection supported)
* `PoetProtocol` (Registrar mock assigns verifiable waits; HMAC proof)
* `PobProtocol` (burn strategy → weighted selection → reward)

All protocols emit `RoundUpdate` with fields minimal for charts:
`{ winner, wait_ms?, votes?, total_burn?, stake_after?, orphaned? }`

## 6) REST/Minimal API (v1)

### Start Simulation

```
POST /api/v1/simulations/start
Content-Type: application/json
{
  "protocol": "poet",
  "nodeCount": 6,
  "rounds": 20,
  "params": { "waitLowMs": 50, "waitHighMs": 300, "seed": 7 }
}
→ 202 Accepted
{ "simulationId": "guid", "hubUrl": "/hubs/sim/..." }
```

### Get Analytics

```
GET /api/v1/simulations/{id}/analytics
→ 200
{
  "height": 20,
  "winnerDistribution": { "N0":3, "N1":4, ... },
  "protocolMetrics": { "avgWaitMs": 124, "quorum": 5, "forks": 0 }
}
```

### List Blocks / Get Block

```
GET /api/v1/blocks?simId={id}&page=1&pageSize=50
GET /api/v1/blocks/{blockId}
```

### Admin: Manage Protocol Configs

```
PUT /api/v1/settings/protocols/poa
{ "authorities": ["N0","N1","N2"] }
```

## 17) Threat Model (short)

* **Replay/Lie about wait** → PoET HMAC proof check.
* **Vote manipulation (PBFT)** → require 2/3 quorum; visualize faulty nodes.
* **Stake/burn grinding** → persistent state & randomness seed; cap per-round changes.
* **DoS** → rate limits and max concurrent runs (1 per user).

## 20) Glossary

* **Round**: one consensus iteration that can produce one block.
* **Winner**: node selected to propose/commit block in a round.
* **Payload Modes**: domain data stored in block (Supply-Chain / FL).
* **Quorum**: minimum “yes” votes to finalize (PBFT).
* **Attestation**: PoET proof a node waited its assigned time.
